Step 1: Exam-to-Level Mapping

Map exam papers to curriculum levels. Each level can have up to 5 different exam versions.

What is Exam-to-Level Mapping?

This is the first configuration step. You need to specify which exam papers are available for each curriculum level.

Key Points:

  • Each curriculum level can have multiple exam versions
  • When a student is placed in a level, the system will randomly select one of the mapped exams
  • The same exam can be mapped to multiple curriculum levels (e.g., Exam C can be used for both Core Phonics Level 1 and Core Phonics Level 2)
  • Use the "+" button to add more exam slots and "Clear" button to remove exams

Step 2: Placement Rules

Configure which curriculum level each student should be placed into based on their grade and English class rank.

What are Placement Rules?

Placement Rules determine which curriculum level a student is assigned to based on their grade and English class rank.

How it works:

  • Each cell in the matrix represents a combination of student grade and English class rank
  • Select the appropriate curriculum level for each combination
  • When a student takes a placement test, the system will use these rules to determine their curriculum level
  • Once the curriculum level is determined, the system will randomly select one of the exams mapped to that level
Example: If you set Primary 3 + Top 10% → PRIME CORE SIGMA - Level 1, then any Primary 3 student in the top 10% of their English class will be placed in PRIME CORE SIGMA - Level 1.
